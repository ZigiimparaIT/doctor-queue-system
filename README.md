# Doctor Queue System

A real-time patient queue management system built with Flutter Web and Firebase. 
This application allows my town medical practice to manage patient queues efficiently through a simple and responsive web interface. Doctors can securely update the current queue number, while patients instantly see the latest number being served in real time.

## Features

- Secure authentication for doctors
- Real-time queue updates powered by Firebase Realtime Database
- Live display of the current queue number
- Displays the time of the last called patient
- Responsive web interface

## Built With

- Flutter Web
- Dart
- Firebase Authentication
- Firebase Realtime Database
- Firebase Hosting

## Why I built this

I built this project because the queue at my local doctor's office was often very long, and patients had no way of knowing when it was their turn unless they stayed inside the waiting room.

The practice already had a ticket machine and a display showing the current number, but both were only visible from inside the building. 
As a result people would queue twice: first, early in the morning to get one of the limited tickets available for the day and then again inside the waiting room to avoid missing their turn.

This defeated the purpose of the ticket system, which was originally introduced to reduce crowding and prevent too many sick patients from gathering in a small waiting area.

I decided to build a web-based solution that allows patients to check the current queue number from anywhere using their phone, while giving the doctor a simple interface to manage the queue in real time.
